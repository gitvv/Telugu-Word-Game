Step 2 — Scroll fix in TeluguKeyboard.tsx.

The modifier shelf auto-scrolls when a consonant is 
tapped. The scroll target is currently hardcoded as 
16 + 12 * 50, which assumes exactly 12 independent 
vowels. This is now wrong — there are 13 items on 
the left side after adding anusvara.

Replace the hardcoded calculation with a dynamic one: 
compute the scroll target from the actual count of 
items before the matras section in the shelf data. 
The formula should read from the data, not assume 
a fixed number.

The scroll behaviour after this fix: when a consonant 
is tapped, Row 1 auto-scrolls so that anusvara (ం) 
appears as the leftmost visible item, with matras 
immediately to the right.

Step 3 — క్ష tap handling.

క్ష is a multi-character Unicode string (క + ్ + ష), 
not a single character. When the player taps the 
క్ష key, route it through the existing onCluster 
callback — the same path Row 2 cluster chips use. 
This commits క్ష as a complete pre-built akshara 
in one state update, without the builder trying to 
parse it as a single Unicode consonant.

One tap on క్ష = one complete akshara committed. 
No halant, no Row 2 involvement.
